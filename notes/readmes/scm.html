GIT

  • Setup
    • Download & install GIT client
    • Set global configuration
      • git config --global user.name "Your Name"
      • git config --global user.email ajcpost@gmail.com
    • Set SSH keys
      • http://help.github.com/win-set-up-git/
      • Add the id_rsa.pub to git website
    • Set remote settings
      • Windows: Use GUI client and set below repository
      • Unix: git remote add origin <repo_name>
    • Repository: git@github.com:ajcpost/ajc.git
    • If creating a new repository
      • mkdir <top_level_dir>
      • cd <top_level_dir>
      • git init
      • add <files>
      • use git commands below
    • Else
      • If git repository is already cloned in the local machine
        • Start using git commands below
      • Else, clone git repository
        • Windows: Use GUI client
  • Commands
    • git status
    • git add *
    • git commit -a -m <update comment>
    • git push origin master

Maven

  • mvn compile will only compile but not create war/ear... use mvn install
  • POM structuring
    • Parent POM defined repositories, properties, dependencies, plugins can be inherited by child pom.
    • Parent POM need not do actual build but rather define individual modules (child poms) for the build.
    • Adhere to standard Maven dir structure (workding_dir/src/main/). Maven is pretty picky about this.
    • Don't use project.name, is a reserved tag?
  • Profiles
    • Can be used to override various properties depending on the build environment (test, dev, production).
      • Deployment path on Unix, Windows may be different.
      • Versions of libraries may be different for dev vs production.
    • Activation tag can be used to activate a profile automatically if certain prerequisites are met.
  • Plugins
    • http://maven.apache.org/plugins/maven-dependency-plugin/ ## unpack contents from dependencies
    • http://mojo.codehaus.org/cobertura-maven-plugin/ ## for unit test coverage reports
